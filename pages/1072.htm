#1072. 「NOIP 2011 Day 1」铺地毯

内存限制：256 MiB 时间限制：1000 ms 输入文件: carpet.in 输出文件: carpet.out
题目类型：传统 评测方式：文本比较
上传者： root
提交 提交记录 统计 测试数据 讨论

题目描述

为了准备一个独特的颁奖典礼，组织者在会场的一片矩形区域（可看做是平面直角坐标系的第一象限）铺上一些矩形地毯。

一共有 nnn 张地毯，编号从 111 到 nnn。现在将这些地毯按照编号从小到大的顺序平行于坐标轴先后铺设，后铺的地毯覆盖在前面已经铺好的地毯之上。地毯铺设完成后，组织者想知道覆盖地面某个点的最上面的那张地毯的编号。

注意：在矩形地毯边界和四个顶点上的点也算被地毯覆盖。

输入

  • 第一行，一个整数 nnn，表示总共有 nnn 张地毯。
  • 接下来的 nnn 行中，第 i+1i + 1i+1 行表示编号 iii 的地毯的信息，包含四个正整数 aaa，bbb，ggg，kkk，每两个整数之间用一个空格隔开，分别表示铺设地毯的左下角的坐标 (a,b)(a, b)(a,b) 以及地毯在 xxx 轴和 yyy 轴方向的长度。
  • 第 n+2n + 2n+2 行包含两个正整数 xxx 和 yyy，表示所求的地面的点的坐标 (x,y)(x, y)(x,y)。

输出

  • 输出共 111 行，一个整数，表示所求的地毯的编号；若此处没有被地毯覆盖则输出 −1-1−1。

样例一

input

3
1 0 2 3
0 2 3 3
2 1 3 3
2 2

output

3

explanation

如下图，111 号地毯用实线表示，222 号地毯用虚线表示，333 号用双实线表示，覆盖点 (2,2)(2, 2)(2,2) 的最上面一张地毯是 333 号地毯。

样例二

input

3
1 0 2 3
0 2 3 3
2 1 3 3
4 5

output

-1

explanation

如上图，111 号地毯用实线表示，222 号地毯用虚线表示，333 号用双实线表示，点 (4,5)(4, 5)(4,5) 没有被地毯覆盖，所以输出 −1-1−1。

限制与约定

  • 对于 30% 的数据，有 n≤2n \leq 2n≤2；
  • 对于 50% 的数据，有 0≤a,b,g,k≤1000 \leq a, b, g, k \leq 1000≤a,b,g,k≤100；
  • 对于 100% 的数据，有 0≤n≤10 0000 \leq n \leq 10\,0000≤n≤10000，0≤a,b,g,k≤100 0000 \leq a, b, g, k \leq 100\,0000≤a,b,g,k≤100000。
