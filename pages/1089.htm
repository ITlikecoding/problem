#1089. 「NOIP 2011 普及组」第二题 统计单词数

内存限制：128 MiB 时间限制：1000 ms 输入文件: stat.in 输出文件: stat.out
题目类型：传统 评测方式：文本比较
上传者： root
提交 提交记录 统计 测试数据 讨论

题目描述

一般的文本编辑器都有查找单词的功能，该功能可以快速定位特定单词在文章中的位置，有的还能统计出特定单词在文章中出现的次数。

现在，请你编程实现这一功能，具体要求是：给定一个单词，请你输出它在给定的文章中出现的次数和第一次出现的位置。注意：匹配单词时，不区分大小写，但要求完全匹配，即给定单词必须与文章中的某一独立单词在不区分大小写的情况下完全相同（参见样例 1），如果给定单词仅是文章中某一单词的一部分则不算匹配（参见样例 2）。

输入格式

输入格式

第 1 行为一个字符串，其中只含字母，表示给定单词；

第 2 行为一个字符串，其中只可能包含字母和空格，表示给定的文章。

输出格式

输出格式

只有一行，如果在文章中找到给定单词则输出两个整数，两个整数之间用一个空格隔开，分别是单词在文章中出现的次数和第一次出现的位置（即在文章中第一次出现时，单词首字母在文章中的位置，位置从 0 开始）；如果单词在文章中没有出现，则直接输出一个整数-1。

样例

输入

input1

To

to be or not to be is a question

input2

to

Did the Ottoman Empire lose its power at that time

输出

output1

2 0

output2

-1

【输入输出样例 2 说明】

表示给定的单词 to 在文章中没有出现，输出整数-1。

数据范围与提示

1≤单词长度≤10。

1≤文章长度≤1,000,000。

解析：我这里采用比较简单的直接以空格区分单词，这样就行了(当然，更复杂的就请自行自行思索，或者问问英语老师，还有什么什么情况能算一个独立单词)
